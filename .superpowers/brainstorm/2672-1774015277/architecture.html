Flusso Esecuzione Agente Singolo

Esempio: /qa-review seo-audit → Marco
1
Login — Playwright naviga a ainstein.it, fa login con credenziali test
2
Review UX/UI — Naviga ogni pagina del modulo, screenshot, analisi visiva (layout, gerarchia, leggibilita, mobile-readiness)
3
Test Funzionali — Esegue i workflow critici (es. crea progetto → importa URL → lancia crawl → legge risultati)
4
Verifica Pattern — Controlla conformita: tabelle CSS, componenti shared, icone Heroicons, lingua IT, landing page pattern
5
Genera Piano Fix — Scrive qa-reviews/{modulo}/{data}.md con issue ordinate per severita + checkbox esecuzione

Flusso Orchestratore

/qa-review-all → Orchestratore
1
Dispatch — Lancia i 6 agenti in parallelo (Agent tool con run_in_background)
2
Attende completamento — Ogni agente scrive il suo file in qa-reviews/
3
Consolida — Legge tutti i piani, genera qa-reviews/_index.md con classifica per score e priorita
