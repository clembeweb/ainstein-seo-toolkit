Post analisi critica: fix concorrenza, scoring, modello + 2 nuovi agenti

Issue risolte dall'analisi critica

✓
Concorrenza limitata

Orchestratore lancia max 2 agenti alla volta (batch da 2) per non sovraccaricare il VPS. Ogni batch attende completamento prima del successivo.

✓
Screenshot in .gitignore

qa-reviews/screenshots/ aggiunto a .gitignore. Sono riferimenti temporanei locali, non vanno nel repo.

✓
Rubrica scoring nella skill

La skill platform-standards include griglia di valutazione: 5 criteri (UX flow, UI polish, pattern compliance, funzionalita, valore aggiunto) con scala 1-10 e pesi. Score finale = media pesata.

✓
Modello Sonnet per tutti gli agenti QA

Analisi visiva di screenshot richiede capacita multimodale forte. Tutti gli agenti QA usano model: sonnet. Pattern auditor puo usare haiku (solo codice, no visual).
