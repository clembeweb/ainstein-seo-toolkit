1.2 Product Description

Ainstein is an AI-powered SaaS platform that provides professional SEO and Google Ads management tools to digital marketing agencies, freelance PPC specialists, and in-house marketing teams. The platform is a modular toolkit where one of the key modules is the Google Ads Analyzer.

The Google Ads Analyzer module enables users to:

  • Analyze Google Ads campaign structures and performance metrics across all campaign types
  • Extract negative keywords using AI-powered analysis of search query reports
  • Evaluate entire ad accounts with multi-level AI insights (account → campaign → ad group)
  • Generate complete Google Ads campaign structures from landing page content
  • Monitor performance metrics (CTR, Quality Score, conversions, ROAS) over time

2. Use Case & Business Need

2.1 Current Situation

We currently use Google Ads Scripts as a data collection mechanism. Users must manually install a JavaScript snippet in their Google Ads account that sends performance data to our platform via HTTP webhook. This approach has significant limitations:

  • Reliability: Scripts are subject to execution time limits and may fail silently
  • User friction: Manual script installation is error-prone and requires technical knowledge
  • Data freshness: Data is only updated when scripts run (scheduled, not real-time)
  • Limited capabilities: Scripts cannot programmatically create campaigns or apply changes
  • Scalability: Each account requires individual script setup and maintenance

2.2 Why We Need Direct API Access

Direct Google Ads API access will allow us to:

  • OAuth-based connection: Users connect their Google Ads account with one click (no scripts to install)
  • Real-time data retrieval: Fetch campaign data on-demand rather than waiting for script executions
  • Campaign creation: Programmatically create campaigns, ad groups, keywords, and ads directly in Google Ads
  • Negative keyword application: Apply AI-recommended negative keywords directly to campaigns
  • Multi-account management: Agency clients can manage multiple Google Ads accounts from a single dashboard
Summary: We need Basic Access to transition from a Google Ads Scripts workaround to proper API integration, providing our agency clients with reliable, real-time campaign management capabilities.

5.2 Campaign Analysis Flow

  1. User selects a connected Google Ads account/project
  2. System fetches campaign data via GoogleAdsService.search() with GAQL queries
  3. Data is stored locally in MySQL (ga_campaigns, ga_ad_groups, ga_ads, ga_ad_group_keywords, ga_search_terms)
  4. User triggers AI evaluation → Claude API analyzes campaign structure and performance
  5. AI generates actionable insights: negative keywords, bid adjustments, ad copy improvements
  6. User can apply negative keywords directly via API mutation

5.3 Campaign Creator Flow

  1. User provides a landing page URL
  2. System scrapes and analyzes the landing page content
  3. AI generates keyword suggestions grouped by ad groups
  4. User reviews and customizes the campaign structure
  5. AI generates responsive search ad copy (headlines, descriptions, display paths)
  6. System creates the complete campaign via Google Ads API mutations:
    • Campaign with budget and bidding strategy
    • Ad groups with targeted keywords (exact, phrase, broad match)
    • Responsive search ads with multiple headlines and descriptions
    • Extensions (sitelinks, callouts, structured snippets)
  7. User can also export to Google Ads Editor CSV format as alternative

6.2 Security Measures

  • HTTPS only: All communication encrypted via TLS 1.3
  • OAuth 2.0: Standard Google OAuth flow, tokens never exposed to frontend
  • CSRF protection: All POST requests require valid CSRF token
  • Prepared statements: All database queries use parameterized queries (no SQL injection)
  • Access control: Project-based isolation; users can only access their own data
  • API keys in database: Never stored in code files or environment variables
  • Session security: HTTP-only cookies, secure flag, session regeneration
  • Data isolation: Multi-tenant architecture with user_id-scoped queries

6.3 Data Sharing Policy

Google Ads data is never shared between different user accounts. Each user's data is isolated and only accessible by the account owner and users explicitly granted access via the project sharing feature (with role-based permissions: Owner, Editor, Viewer).

Google Ads data is not sold, shared with third parties, or used for advertising purposes. Data is only used to provide the analysis and management features described in this document.

7. Rate Limiting & Compliance

7.1 API Request Volume Estimates

Campaign mutations50-200 requestsCampaign creator only
Keyword mutations100-500 requestsNegative keyword application
Total estimated750-3,200 requests/day

7.2 Rate Limiting Strategy

  • Request throttling: Maximum 1 request per second per user account
  • Batch operations: Group mutations into batch requests where possible
  • Caching: Campaign data cached locally, refreshed on-demand or via daily sync
  • Exponential backoff: Automatic retry with exponential backoff on rate limit errors
  • Daily quota monitoring: Track API usage per user and enforce per-user daily limits

7.3 Compliance

  • Full compliance with Google Ads API Terms and Conditions
  • Full compliance with Developer Token policies
  • No automated bidding or budget changes without explicit user confirmation
  • No scraping or crawling of Google properties
  • GDPR compliant data handling (EU-based platform)
  • Users can disconnect their Google Ads account and delete all associated data at any time

8.2 OAuth Flow

  1. User clicks "Connect Google Ads" in their Ainstein project dashboard
  2. System redirects to Google OAuth consent screen with adwords scope
  3. User grants permission and is redirected back to Ainstein
  4. System exchanges authorization code for access + refresh tokens
  5. Tokens are stored encrypted in database, associated with the user's project
  6. All subsequent API calls use the stored tokens with automatic refresh
